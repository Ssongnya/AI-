# 데이터 생성 방법

## 1. 합성데이터(Synthetic Data)

### (1) 정의
- 실제 세계에 존재하지 않는 데이터를 **인공적으로 생성한 데이터**
- 현실의 제약(비용, 시간, 윤리 등) 없이 다양한 데이터를 확보할 수 있는 방법

### (2) 목적
- 데이터 부족 문제를 해결하여 **학습 데이터의 다양성과 양을 확대**
- 희귀하거나 수집이 어려운 상황(예: 자율주행 사고, 의료영상 등)을 **가상 환경에서 재현**
- 실제 데이터와 유사한 패턴을 가지도록 설계되어 **모델의 일반화 성능 향상**에 기여

### (3) 예시
- 3D 모델, 시뮬레이션, 이미지 합성, 생성형 AI(GAN, Diffusion 모델 등)를 이용하여  
  존재하지 않는 이미지를 새로 만들어내는 경우


## 2. 데이터 증강(Data Augmentation)

### (1) 정의
- 기존의 실제 데이터를 변형하여 **추가적인 학습 데이터 세트를 확보하는 방법**
- 새로운 데이터를 생성하지 않고, **기존 데이터를 다양하게 변화시켜 학습 성능을 향상**

### (2) 방법
- 이미지의 경우 다음과 같은 변형이 주로 사용됨:
  - 회전 (Rotation)
  - 이동 (Translation)
  - 좌우 반전 (Flipping)
  - 크기 조정 (Scaling)
  - 색상·밝기 변화 (Color jittering)
  - 노이즈 추가 (Noise injection)

### (3) 효과
- **데이터 편향(Bias)** 을 줄여 모델의 일반화 능력 향상  
- **오버피팅(Overfitting)** 완화 — 학습 데이터가 다양해지므로 모델이 특정 패턴에 과적합되지 않음  
- **데이터 효율 향상** — 적은 원본 데이터로도 충분히 학습 가능

## 3. 합성데이터 vs 데이터 증강 비교

| 구분 | 합성데이터 (Synthetic Data) | 데이터 증강 (Data Augmentation) |
|------|------------------------------|--------------------------------|
| 생성 방식 | 완전히 새로운 데이터 생성 | 기존 데이터 변형 |
| 기반 데이터 | 존재하지 않음 (가상 생성) | 실제 데이터 기반 |
| 목적 | 데이터 부족 문제 해결 | 데이터 다양성 확보 및 과적합 방지 |
| 활용 예시 | 3D 시뮬레이션, GAN 생성 이미지 | 회전, 반전, 색상 변화 등 |
| 장점 | 새로운 상황 시뮬레이션 가능 | 간단하고 계산 비용 낮음 |
| 단점 | 생성 품질에 따라 학습 영향 큼 | 변화가 제한적일 수 있음 |


## 4. 핵심 정리
- **합성데이터(Synthetic Data)**  
  → 실제 존재하지 않는 데이터를 새롭게 생성  
  → 모델이 다양한 상황을 학습하도록 돕는 인공 데이터

- **데이터 증강(Data Augmentation)**  
  → 실제 데이터를 변형하여 데이터셋을 확장  
  → 학습 데이터의 다양성을 높여 모델의 일반화 성능 강화

- 두 방법 모두 **데이터 생성 방법(Data Generation Method)** 에 속하며,  
  **학습 효율 향상과 일반화 성능 개선**을 위한 필수 전략임.
