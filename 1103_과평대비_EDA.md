# EDA (Exploratory Data Analysis, 탐색적 데이터 분석)

## 1. EDA의 개념

### (1) 정의
- **Exploratory Data Analysis (EDA)**  
  → 데이터를 시각화하거나 통계적으로 탐색하여 **데이터의 특성과 구조를 이해하는 과정**  
- 모델링 전에 반드시 수행되는 필수 단계로,  
  **데이터 품질, 이상치, 분포, 변수 간 관계**를 파악하기 위함이다.

### (2) 목적
- 데이터의 **분포와 패턴**을 시각적으로 이해  
- **이상치(Outlier)** 및 결측치(Missing Value) 탐지  
- 변수 간 **상관관계(Correlation)** 파악  
- 분석 방향 및 모델링 전략 수립


## 2. EDA 주요 단계

### (1) 데이터 구조 및 샘플 확인
- 데이터의 **형태(행/열), 변수 이름, 타입** 등을 살펴봄  
- 실제 값 몇 개를 출력해 **데이터 품질과 특징을 파악**  
- 예: 숫자형/범주형 변수 구분, 누락 데이터 존재 여부

### (2) 기술통계 요약 (Descriptive Statistics)
- 평균, 중앙값, 최댓값, 표준편차 등의 **기본 통계량**을 확인  
- 각 변수의 데이터 범위와 변동성을 이해  
- 예: “평균보다 극단적으로 큰 값이 많다 → 분포의 비대칭성 존재”

### (3) 데이터 분포 확인 (Distribution Analysis)
- 히스토그램(Histogram), 박스플롯(Boxplot) 등을 통해 변수의 **분포 형태** 파악  
- 정규분포 여부, 이상치의 존재 등을 시각적으로 탐색  
- 예: 자동차 연비(mpg)의 분포, 가중치(weight)의 범위 확인

### (4) 변수 간 관계 확인 (Relationship Analysis)
- 산점도(Scatter plot), 페어플롯(Pair plot) 등으로 변수 간 관계 시각화  
- 두 변수 간의 **상관성**(positive/negative correlation) 파악  
- 예: 자동차 무게(weight) 증가 → 연비(mpg) 감소 (음의 상관관계)


## 3. EDA에서 자주 사용하는 시각화 기법

| 시각화 기법 | 설명 | 활용 예시 |
|--------------|------|-----------|
| **히스토그램 (Histogram)** | 데이터의 분포와 빈도 확인 | 연비, 가격 등의 분포 확인 |
| **박스플롯 (Boxplot)** | 이상치 및 중앙값 확인 | 지역별 소득 비교 |
| **산점도 (Scatter Plot)** | 두 변수 간 관계 확인 | 무게 vs 연비 관계 |
| **페어플롯 (Pair Plot)** | 여러 변수 간 상관 구조를 한눈에 확인 | 자동차 성능 변수 비교 |
| **히트맵 (Heatmap)** | 변수 간 상관계수를 색상으로 시각화 | 다중 변수 간 상관 정도 확인 |


## 4. 상관관계 분석 (Correlation Analysis)

### (1) 상관계수(Correlation Coefficient)
- 두 변수 간의 선형 관계를 수치로 표현한 값  
- 값의 범위: **-1 ≤ r ≤ 1**
  - **r > 0** → 양의 상관관계 (값이 함께 증가)  
  - **r < 0** → 음의 상관관계 (한쪽 증가 시 다른 쪽 감소)  
  - **r ≈ 0** → 선형 관계 없음  

### (2) 다중상관관계 분석
- 여러 변수들 간의 상관성을 동시에 분석  
- **Heatmap**으로 각 변수 쌍의 상관 정도를 시각화  
- 높은 상관을 가진 변수는 **다중공선성(multicollinearity)** 문제를 일으킬 수 있으므로 모델링 전 확인 필요


## 5. EDA를 통해 확인해야 할 항목 정리

| 구분 | 주요 내용 |
|------|-----------|
| **데이터 구조 확인** | 데이터 크기(shape), 컬럼명, 타입 |
| **기초 통계량 확인** | 평균, 중앙값, 표준편차 등 |
| **결측치 확인** | NaN, Null 값의 개수 및 위치 |
| **이상치 확인** | Boxplot으로 범위 밖 데이터 확인 |
| **분포 시각화** | Histogram, KDE plot |
| **변수 간 관계 확인** | Scatter plot, Pair plot |
| **상관계수 분석** | Heatmap, `corr()` 결과 비교 |


## 6. EDA의 역할과 중요성

### (1) 모델링 이전의 “데이터 탐색 단계”
- 데이터의 품질을 보정하고, 어떤 전처리(정규화, 인코딩 등)가 필요한지 결정  
- 학습 전에 “데이터가 말하는 패턴”을 이해할 수 있게 해줌  

### (2) 문제 해결 방향 설정
- 단순히 시각화가 아니라, **데이터의 스토리**를 발견하는 과정  
- 예: “차량 무게가 늘어날수록 연비가 낮다” → 에너지 효율 모델링 시 변수 후보로 활용 가능


## 7. 한눈에 보는 EDA 핵심 요약

| 단계 | 주요 목표 | 대표 도구 / 방법 |
|-------|------------|------------------|
| 데이터 확인 | 구조/형태 파악 | `head()`, `info()`, `describe()` |
| 통계량 요약 | 범위 및 중심 경향성 | `mean()`, `median()`, `std()` |
| 분포 시각화 | 변수별 형태 파악 | Histogram, Boxplot |
| 관계 분석 | 변수 간 영향 탐색 | Scatterplot, Pairplot |
| 상관분석 | 다중 변수 관계 | `corr()`, Heatmap |

---

✅ **핵심 요약**
> EDA는 데이터 분석의 출발점이다.  
> 데이터를 “그대로 믿기 전에” 시각화와 통계를 통해  
> 분포, 이상치, 상관관계를 이해함으로써  
> **모델링의 방향성과 품질을 결정하는 가장 중요한 과정**이다.
